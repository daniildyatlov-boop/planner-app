# Wireframe документация: Персональный планер

## Общая концепция

Низкоуровневый wireframe для gesture-first интерфейса планера в Telegram Mini App.

### Принципы дизайна
- Только оттенки серого
- Системный шрифт
- Без кнопок, таббаров, навигации
- Фокус на жестах и зонах взаимодействия
- Намеренно "незаконченный" вид

## Основной экран: ДЕНЬ

### Структура
```
[Зона добавления - pull down]
[Заголовок дня]
[Планы с временем]
[Планы без времени]
[Зоны смены дня - swipe left/right]
```

### Жесты на экране "День"

| Жест | Действие | Зона |
|------|----------|------|
| Pull down | Добавить новый план | Верхняя часть экрана |
| Swipe left | Предыдущий день | Левая граница экрана |
| Swipe right | Следующий день | Правая граница экрана |
| Swipe по плану | Удалить план | Правая часть элемента плана |
| Tap по плану | Фокус/детали плана | Основная область плана |
| Long tap по плану | Редактирование плана | Основная область плана |
| Tap по комнате | Фильтр по комнате | Бейдж комнаты |

### Состояния экрана

1. **Пустой день**
   - Центральное сообщение о пустом дне
   - Подсказка о добавлении плана
   - Все жестовые зоны активны

2. **Обычный день**
   - 3-5 планов
   - Разделение на планы с временем и без
   - Комнаты показываются как бейджи

3. **Перегруженный день**
   - 8+ планов
   - Плотная компоновка
   - Сохранение читаемости

## Экран добавления плана

### Структура
```
[Заголовок]
[Текстовое поле ввода]
[Подсказки и примеры]
[Зоны жестов]
```

### Жесты на экране добавления

| Жест | Действие | Зона |
|------|----------|------|
| Enter | Сохранить план | Клавиатура |
| Swipe down | Отмена | Нижняя часть экрана |
| Swipe left | Выбор даты/времени | Левая граница |
| Swipe right | Выбор комнаты | Правая граница |

### Логика распознавания текста

Система автоматически распознает в свободном тексте:
- **Время**: "18:00", "вечером", "утром"
- **Дату**: "завтра", "вторник", "понедельник"
- **Комнату**: последнее слово или контекстное слово

#### Примеры распознавания:
- "Силовая вторник 18:00 зал" → Время: 18:00, День: вторник, Комната: зал
- "Встреча 15:00" → Время: 15:00, День: сегодня, Комната: нет
- "Скраб вечером" → Время: вечер, День: сегодня, Комната: нет
- "Звонок маме завтра утром дом" → Время: утро, День: завтра, Комната: дом

## Комнаты

### Принципы работы с комнатами
- Полностью опциональны
- Создаются автоматически при первом упоминании
- Пользователь называет как хочет
- Отображаются как небольшие бейджи
- Не влияют на основную структуру интерфейса

### Примеры комнат
- "зал", "офис", "дом", "парк", "клиника"
- Любые названия по желанию пользователя

## Зоны жестов

### Визуальное обозначение в wireframe
- Пунктирные границы для зон жестов
- Полупрозрачная заливка для различения типов зон
- Текстовые подписи с описанием действий

### Цветовое кодирование зон (только в wireframe)
- Зеленый оттенок: добавление контента
- Синий оттенок: навигация
- Красный оттенок: удаление/отмена
- Оранжевый оттенок: дополнительные действия

## Технические требования

### Размеры экрана
- Ширина: 375px (iPhone стандарт)
- Высота: 667px минимум
- Адаптивность под разные размеры Telegram

### Отступы и размеры
- Основные отступы: 20px
- Межстрочные интервалы: 1.4
- Размеры шрифтов: 12px-24px
- Высота тапабельных зон: минимум 44px

### Жестовые зоны
- Минимальная ширина для swipe: 30px
- Высота pull-зоны: 100px
- Область для long tap: вся область элемента

## Готовность к визуальному дизайну

Wireframe готов для наложения iOS 26 визуального стиля:
- Четкая иерархия элементов
- Определенные зоны взаимодействия
- Логичная структура контента
- Продуманные жесты и переходы
- Масштабируемая система состояний
